一、什么是Wrap文件
Wrap（包装类） 是 ToLua 自动生成的 C# 静态类，作为 Lua 和 C# 之间的桥梁。每个需要导出到 Lua 的 C# 类都会生成一个对应的 XxxWrap.cs 文件。
┌─────────────┐ ┌────────────────┐ ┌─────────────────┐
│ Lua 脚本 │ ──────▶ │ XxxWrap.cs │ ──────▶ │ 原生 C# 类 │
│ go:SetActive │ │ (自动生成桥梁) │ │ GameObject │
└─────────────┘ └────────────────┘ └─────────────────┘

四、具体Lua调用到C#函数的流程
当 Lua 调用go：SetActive(true)时：

  1. Lua 调用 SetActive
    ↓
  2. 查找 GameObject metatable 的 SetActive 函数
    ↓
  3. 调用 UnityEngine_GameObjectWrap.SetActive(IntPtr L)
    ↓
  4. 从 Lua 栈获取参数:
    • 位置 1: self (GameObject userdata)
    • 位置 2: true (boolean)
      ↓
  5. 通过 ObjectTranslator 将 userdata 转换为 C# GameObject
    ↓
  6. 调用真正的 C# 方法: obj.SetActive(true)
    ↓
  7. 返回结果到 Lua 栈
    第一步到第二步比较好理解，lua调用SetActive的时候首先就是查找GameObject元表里是否有SetActive这个方法，有则调用。而第二步到第三步可能会有点疑问为什么可以直接调用到Wrap文件，要理解这一点需要理解ToLua是如何把Wrap文件注册到Lua表里的。看代码：
